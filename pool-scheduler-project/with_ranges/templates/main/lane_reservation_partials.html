Here we use the __startswith and __endswith lookups along with F Expressions . We are subtracting the lower part of the range (the starting value) from the upper end of the range (the ending value) to annotate each instance in the QuerySet with the overall length of the range. We then use this annotation to filter out any reservations that are not greater than 8 hours in length.

views.py


from django.db.models import DurationField, ExpressionWrapper, F

lane_reservations_greater_than_eight_hr = (
    LaneReservation.objects.all()
    .annotate(
        delta=ExpressionWrapper(
            F("period__endswith") - F("period__startswith"),
            output_field=DurationField(),
        )
    )
    .filter(delta__gt=timezone.timedelta(hours=8))
)

We start with a helper function in date_utils.py, which returns a DateTimeTZRange with a range that covers the entire current week, either starting on Sunday or Monday. DateTimeTZRange comes from psycopg2.extras .

date_utils.py


def get_this_week_range(starting_day_sunday: bool = True) -> DateTimeTZRange:
    """
    Return a DateTimeTZRange range covering the entirety of current week, starting on either Sunday or Monday
    """
    today = timezone.now().date()

    if starting_day_sunday:
        # This section starts the week on Sunday to Saturday
        start_date = today - timezone.timedelta(days=(today.weekday() + 1) % 7)
        end_date = start_date + timezone.timedelta(days=6)

    else:
        # This section starts the week on Monday to Sunday
        start_date = today - timezone.timedelta(days=today.weekday())
        end_date = start_date + timezone.timedelta(days=6)

    return DateTimeTZRange(start_date, end_date)

Then we use the __overlap lookup to filter our QuerySet to include only LaneReservation periods that overlap (any amount) with the current week.

date_utils.py


from django.utils import timezone
from dateutil.relativedelta import relativedelta
from psycopg2.extras import DateTimeTZRange

def get_month_range() -> DateTimeTZRange:
    """
    Return a DateTimeRange range covering the entirety of current month
    """
    today = timezone.now().date()
    if today.day > 25:
        today += timezone.timedelta(7)
    this_month_start = today.replace(day=1)
    next_month_start = this_month_start + relativedelta(months=1)
    return DateTimeTZRange(this_month_start, next_month_start)

Here, we create a form with a single CharField and do the following:

  • add the datetime_rangepicker class, which we will use in the template to attach the daterangepicker
  • add some htmx attributes which will post the input to the correct url when the onchange event fires, and place the response into #outputDiv
  • clean the resulting field data by converting the string to a psycopg2.extras DateTimeTZRange, which can be used in our QuerySet to filter results


class ReservationsOverlappingDateTimeForm(forms.Form):
    datetimerange_input = forms.CharField(
        label="DateTime Range to Filter",
        widget=forms.TextInput(
            attrs={
                "class": "datetime_rangepicker",
                "hx-target": "#outputDiv",
                "hx-trigger": "range_change",
                "onchange": "htmx.trigger(this, 'range_change')",
            }
        ),
    )

    def clean_datetimerange_input(self):
        raw_string_list = self.cleaned_data["datetimerange_input"].split(" - ", 1)
        if len(raw_string_list) != 2:
            raise ValidationError("Incorrect datetime range input!")

        try:
            data = get_datetime_from_string_list(raw_string_list)
        except ValueError as e:
            raise ValidationError(e)

        return data

In the view, a GET request will return the form and an empty #outputDiv div tag. On POST, we filter the QuerySet to only those LaneReservation instances which overlap with the datetime range from user input.

Average Duration of All Reservations

Aggregates the average durations of all LaneRental period durations.
Average: {{ lane_reservations_average_length_of_all }}

We annotate each Lane in the QuerySet with the duration of period, by subtracting the lower value of the DateTimeRangeField from the upper value, and then apply a Sum aggregation on that new annotation. Finally, we convert the number of seconds into a string representation of the resulting annotation and display it to the user.

views.py


lane_reservations_average_length_of_all = LaneReservation.objects.annotate(
    time_elapsed=Sum(Cast(F("period__endswith") - F("period__startswith"), DurationField()))
).aggregate(time_elapsed_avg=Avg("time_elapsed"))

average_time = lane_reservations_average_length_of_all["time_elapsed_avg"].seconds

# Convert overall `timedelta` seconds into minutes, hours, and seconds
hours, remainder = divmod(average_time, 3600)
minutes, seconds = divmod(remainder, 60)
